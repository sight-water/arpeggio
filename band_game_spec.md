# バンド・オーディション — 設計仕様 v2.1

> **正はつねに `band_audition_game.html` の実装**。このファイルは実装の写しであり、実装と矛盾した場合は実装が正しい。

---

## 1. 概要

- 2〜5人。各プレイヤーは「バンドのリーダー兼1枠の演奏者」。
- 各自 **14手番**。手番ごとに **練習** か **スカウト** を1つ行う。
- 全員の手番終了後、**2曲のオーディション**で順位点を競い、合計点が最多のプレイヤーが優勝。

---

## 2. トークン経済

### 共有トークンプール
- 場に **縦トークン** と **横トークン** を一山ずつ置く。
- プールサイズ：`max(20, 人数 × 4 + 6)` 枚ずつ（CONFIG で変更可）。
- スカウトで支払ったトークンはプールに戻る（流動する有限資源）。

### 練習（メインアクション）
- 縦か横を選び、共有プールから **2トークン** を取って手元に置く。
- 手元のトークンは **そのままその軸のパワー** として機能（消費しない）。
- プールが1枚しかない場合は1枚だけ取る（0のときは選択不可）。
- 高さは練習で伸ばせない（高さトークンは存在しない）。

### スカウト（メインアクション）
- 場（マーケット）のメンバー1人を選び、**楽器ごとのコストをトークンで支払う**。
- 支払ったトークンはプールに戻る。
- メンバーがバンドに加わり、クローズドスキルが公開される（スカウト公開演出あり）。
- バンド上限：**6人（自分含む）**。

### パス（緊急手段）
- 両プールが0かつスカウト可能なメンバーもいない場合のみ選択できる。
- 手番を消費するだけで何も得られない。ゲーム上はほぼ発生しない想定。

### スカウトコスト表

| 楽器 | コスト |
|---|---|
| ボーカル | 横×4 |
| ギター | 縦×1＋横×4 |
| ベース | 縦×3 |
| カホン | 縦×4 |
| キーボード | 縦×2＋横×2 |

## 3. 楽器の基礎値

| 楽器 | 縦 | 横 | 高さ | 担当軸 |
|---|---|---|---|---|
| ボーカル | 0 | 3 | 1 | 横 |
| ギター | 1 | 3 | 1 | 横 |
| ベース | 2 | 0 | 1 | 縦 |
| カホン | 3 | 0 | 1 | 縦 |
| キーボード | 0 | 0 | 3 | 高さ |

---

## 4. パワーの計算

```
縦パワー = バンド全員の縦基礎値合計 + 手元の縦トークン + スキル補正（縦）
横パワー = バンド全員の横基礎値合計 + 手元の横トークン + スキル補正（横）
高さパワー = バンド全員の高さ基礎値合計 + スキル補正（高さ）  ※トークンなし
```

---

## 5. 手番の流れ

- 時計回り。各自14手番。
- 手番に **練習** か **スカウト**（か合法手なし時のパス）を1つ。
- 加えて **予約** と **覗き** を任意で使える（手番を消費しない）。

### 予約（未実装・v3候補）
- 仕様上は「場のメンバー1人を予約。自分以外はスカウトできなくなる」だが、現実装には含まれない。

### 覗き（ゲーム中1回・手番消費なし）
- 「👁 覗き（残1回）」ボタンから覗きモードに入る。
- 覗きモード中に市場のメンバーを1人タップすると、そのメンバーの **クローズドスキルが自分だけに公開** される。
- カード上に `👁 隠: スキル名` と表示され、リスクスキルなら赤枠に変わる。
- 使用後はボタンが「覗き使用済」に変わり再使用不可。
- 手番消費なし。覗いた後に通常の練習・スカウトを行ってターンを終える。

### マーケット補充
- スカウト後、場を `人数+2` 枚に補充する（山札から即補充）。

## 6. スキル（全7種）

メンバーは **オープンスキル1枚（表向き）** と **クローズドスキル1枚（裏向き）** をランダムに持つ。プレイヤー自身はスキルを持たない（楽器のみ）。

| スキル名 | 効果 | リスク |
|---|---|---|
| ハモり上手（harmonizer） | 高さ+2 | なし |
| 大編成向き（big_ensemble） | バンドが5人以上なら横+3 | なし |
| 几帳面なリズム（tight_rhythm） | バンドにカホン/ベースがいれば縦+2 | なし |
| 隠れた名手（hidden_ace） | このメンバーの担当軸+2 | なし |
| ストイック（stoic） | バンドが3人以下ならこのメンバーの担当軸+4 | なし |
| 本番×（stage_fright） | 各曲70%の確率で縦−2・横−2 | **あり** |
| はみ出し者（misfit） | バンドが4人以上ならこのメンバーの担当軸−2 | **あり** |

**覗き設計意図**：クローズドが当たりかリスクかはスカウトするまで不明。覗き（1回限り）でスカウト前に確認できる。「賭けてスカウトするか、覗いてから判断するか」が中核の駆け引き。

## 7. オーディション（全手番終了後）

### 曲のスコア式

- **Song1（バラード）** ＝ `高さ×2 + 横×1`
- **Song2（アップテンポ）** ＝ `縦×2 + 横×1`

### 順位点

各曲について全バンドをスコア順に並べ、順位点を配る：

| 人数 | 順位点（1位→最下位） |
|---|---|
| 2 | 3, 0 |
| 3 | 5, 2, 0 |
| 4 | 5, 3, 1, 0 |
| 5 | 5, 3, 2, 1, 0 |

### 審査員ボーナス
セットアップ時に審査員カードを2枚公開。最終的な合計順位点に上乗せされる。

### 勝利条件
2曲の順位点合計＋審査員ボーナスの **合計が最多** のプレイヤーが優勝。  
同点時：横パワーが高い方。それも同じ場合はランダム（シード付き）。

## 8. 審査員カード（全6種、毎ゲーム2枚公開）

| カード名 | 効果 | ボーナス |
|---|---|---|
| リズム重視（rhythm_lover） | Song2 の1位バンドに | +1pt |
| 大バンド好き（big_band） | メンバー数最多バンドに | +2pt |
| サプライズ枠（surprise） | 楽器種類数が最多バンドに | +2pt |
| 実力主義（hard_work） | 練習回数が最多プレイヤーに | +1pt |
| ボーカル推し（vocal_fan） | ボーカルを含むバンドに | +1pt |
| ハーモニー好き（harmony_fan） | Song1 の1位バンドに | +1pt |

## 10. bot 戦略アーキタイプ（参考）

- **WideBot**：多様な楽器を集めて高さを稼ぎ Song1 狙い。横を練習。
- **NarrowBot**：カホン/ベース中心に少人数。縦を練習して Song2 狙い。
- **BalancedBot**：4人前後、縦横を分散練習。両曲で2位を狙う。
- **RandomBot**：合法手をランダム選択のベースライン。
