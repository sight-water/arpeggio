# バンド・オーディション — 開発依頼（Claude Code 用）

Python で、自作ボードゲーム「バンド・オーディション（仮）」を **プレイ＆バランス調整** するためのプログラムを作ってほしい。
目的は「遊びながら数値を調整する」こと。なので **調整したい数値はすべて config に集約** し、ロジックは UI から切り離すこと。下記の数値が唯一の正（`rules.html` は人間用の要約で、数値はこのファイルが優先）。

---

## 1. アーキテクチャ（必須）

```
band_game/
  engine/        … ルールだけの純粋ロジック。print/input を一切持たない。テスト可能に
  config.py      … 調整したい数値を全部ここに。コードを触らず数値だけで調整できること
  skills.py      … スキル/カードを「データ＋効果フック」で定義（追加・削除・改変が容易な registry 方式）
  bots.py        … 戦略botの実装
  play.py        … ターミナルで2〜5人ホットシート対戦できる対話モード
  simulate.py    … botでN回自動対戦し、戦略ごとの勝率などを集計
  tests/         … スコア計算・種類数ボーナス・順位点の単体テスト
  README.md
```

- Python 3.10+、**標準ライブラリのみ**（`random`, `dataclasses`, `typing`, `argparse`, `statistics` など）。外部依存なし。
- エンティティは dataclass、型ヒントと docstring を付ける。短く読みやすく。
- **スキルはハードコードせず、データ＋小さな効果関数（フック）で表現**。後で頻繁に足し引きするため。

---

## 2. ゲーム概要

- 2〜5人。各プレイヤーは「バンドのリーダー兼1枠の演奏者」。
- 各自 **合計8手番**。手番ごとに **スカウト** か **練習** を1つ行う。
- 全員の手番終了後、**2曲のオーディション**で順位点を競い、合計点が最多のプレイヤーが優勝。

### プレイヤーが持つもの
- 楽器1つ（バンドの1枠として基礎値を出す）
- リーダースキル1つ
- オープンスキル1つ
（プレイヤー自身にクローズドスキルは無い）

### メンバーが持つもの
- 楽器1つ（基礎値を出す）
- オープンスキル1つ（表向き＝最初から見える）
- クローズドスキル1つ（裏向き＝スカウトして初めて公開）

---

## 3. 軸と楽器の基礎値

軸は3つ：**rhythm（縦）/ melody（横）/ harmony（高さ）**。

楽器の基礎値（バンドにいる間、各軸に加算）：

| instrument | rhythm | melody | harmony |
|---|---|---|---|
| vocal    | 0 | 3 | 1 |
| guitar   | 1 | 3 | 0 |
| bass     | 2 | 0 | 0 |
| cajon    | 3 | 0 | 0 |
| keyboard | 0 | 0 | 3 |

**担当軸（primary axis）**：rhythm = cajon, bass ／ melody = vocal, guitar ／ harmony = keyboard。
（「担当軸 +N」系のスキルは、そのメンバー自身の楽器の担当軸に作用する）

同じ楽器の重複スカウトは可（基礎値は重ねて加算されるが、種類数は増えない）。

---

## 4. パワーの計算

各軸のパワー ＝
`バンド全員（プレイヤー含む）の基礎値の合計` ＋ `その軸への練習の合計` ＋ `スキル補正` ＋（harmony のみ）`種類数ボーナス`

- **種類数ボーナス（harmony専用）**：`harmony += 2 × max(0, distinct_instrument_types − 2)`
  - 3種で+2、4種で+4、5種で+6。`飛び道具`スキルは distinct カウントを +1 する。
- **練習**：1回につき選んだ軸に **+2**。対象は **rhythm か melody のみ**。**harmony は練習で伸ばせない**。

---

## 5. 手番の流れ

- 時計回り。各自8手番。
- 手番に **スカウト** か **練習** を1つ。加えて **妨害カードを1枚まで**使える（任意・手番の行動を消費しない）。
  - **スカウト**：場（market）のメンバー1人をバンドに加える → そのクローズドを公開 → 場を `人数+2` 枚に補充。**バンド上限6人（自分含む）**。
  - **練習**：rhythm か melody に +2。

### セットアップ
1. 各自、楽器を1つ選ぶ。
2. リーダースキルを2枚引き1枚を残す。オープンスキルを1枚引いて自分に付ける。
3. メンバー山札をシャッフル。場に `人数+2` 枚を表向きに並べる（各メンバーはオープン表・クローズド裏）。
4. 各自、妨害カードを2枚引く。
5. パワートラック（rhythm/melody/harmony）を 0 に。

---

## 6. オーディション（全手番終了後）

2曲を演奏。各曲のスコア：
- **Song1（ballad）** ＝ `harmony×2 + melody×1`
- **Song2（uptempo）** ＝ `rhythm×2 + melody×1`

各曲について全バンドをスコア順に並べ、**順位点**を配る（config 化）。デフォルト：

| 人数 | 順位点（1位→最下位） |
|---|---|
| 2 | 3, 0 |
| 3 | 5, 2, 0 |
| 4 | 5, 3, 1, 0 |
| 5 | 5, 3, 2, 1, 0 |

2曲の順位点の **合計が最多** のプレイヤーが優勝。
同点 → **アンコール**：melody が高い方が勝ち。melody も同点なら config の tie_breaker（デフォルト：シード付き乱数で1人に決定。人間プレイ時は同時優勝でも可）。

---

## 7. 妨害カード（各自2枚、手番に1枚まで、使用後は捨てる）

- **studio_lockout（スタジオ確保）**：対象プレイヤー1人の「次の練習」を +1 に弱める（通常+2）。
- **snipe（横取り宣言）**：場のメンバー1人に予約。使用者の次の手番まで、使用者しかスカウトできない。
- **backstage_peek（楽屋のぞき見）**：場 or 相手バンドのクローズド1枚を、自分だけ確認。

---

## 8. スキル一覧

補正は基本「スコア計算時に加算」。練習系はその練習時、アンコール系はアンコール時、`clutch`は各曲の最終スコアに加算。複数同種スキルは加算スタック。

### リーダースキル（1人1つ）
- **metronome**：rhythm の練習が +3（通常+2）。
- **frontman**：アンコールの melody 勝負で自分の melody +4。
- **headhunter**：ゲーム中2回、スカウト直前にそのメンバーのクローズドを覗ける（botは「覗いてからリスク回避判断」に使う）。
- **stoic**：本番時、バンドが3人以下なら全軸 +4。

### オープンスキル（表向き）
- **big_ensemble（大編成向き）**：バンドが5人以上なら melody +3。
- **small_elite（少数精鋭）**：バンドが3人以下なら、そのメンバーの担当軸 +3。
- **harmonizer（ハモり上手）**：harmony +2。
- **tight_rhythm（几帳面なリズム）**：バンドに cajon か bass がいれば rhythm +2。
- **crowd_pleaser（ムードメーカー）**：アンコールの melody 勝負で melody +2。
- **wildcard（飛び道具）**：6種目の特殊楽器扱い。distinct_instrument_types を +1（harmony の種類数ボーナスが伸びる）。

### クローズドスキル（裏向き・スカウトで公開）
あたり：
- **hidden_ace（隠れた名手）**：そのメンバーの担当軸の基礎値 +2。
- **clutch（本番に強い）**：各曲のバンド最終スコアに +3。
- **born_star（天性のスター）**：アンコールの melody 勝負で melody +5。

リスク：
- **stage_fright（本番×）**：各曲ごとに **70%の確率**で発動し、その曲のスコア計算で rhythm −2 と melody −2（発動判定はシード付き乱数）。
- **practice_averse（練習嫌い）**：バンドにいる間、練習の加算が −1。
- **misfit（はみ出し者）**：バンドが4人以上だと、そのメンバーの担当軸 −2。

---

## 9. 隠匿情報の扱い（重要）

- bot もクローズドスキルは **スカウトするまで見えない**（人間と同条件）。スカウトすると公開＝確定。地雷（stage_fright等）を引くリスクをそのまま再現すること。
- `headhunter` / `backstage_peek` を使ったときだけ、その1枚を事前に知れる。
- これによりシミュレーションが「実戦の読み合い」を反映したものになる。

---

## 10. bot戦略（bots.py）

ヒューリスティックで良い（最適化は不要、アーキタイプ比較が目的）：
- **WideBot**：種類数を増やす方向に最大6人までスカウト。残り手番は melody を練習。既定リーダー=frontman、楽器=vocal。
- **NarrowBot**：cajon/bass 中心に2人ほどスカウトして3人で止め、rhythm 中心に練習（melodyにも少し）。既定リーダー=metronome、楽器=vocal。
- **BalancedBot**：3軸を埋める4人前後を狙い、練習は rhythm/melody を分散。既定リーダー=stoic か headhunter。
- **RandomBot**：合法手をランダムに選ぶベースライン。
- 妨害カードは「相手の練習直前に studio_lockout」「自分が欲しい場のメンバーに snipe」など簡単な条件で使う実装でよい。

---

## 11. simulate.py の出力

- `--games N`（既定1000）、`--seed S`、参加させる戦略を指定できる。
- 座席バイアス除去のため、戦略の手番順をローテーション or ランダム化。
- 集計して表で表示：戦略ごとの **勝率 / 平均合計点 / Song1平均 / Song2平均 / アンコール到達率 / アンコール勝率**。
- 1つの seed で結果が再現すること（デッキシャッフル・クローズド配布・bot乱数・stage_fright判定すべてを seed 管理）。

---

## 12. 進め方

1. まず `engine` + `config` + `skills` + `bots` + `simulate` を動く形で実装。
2. **WideBot / NarrowBot / BalancedBot で1000回回し、勝率の表を見せて。**（特に「1勝1敗時のアンコールで広い型が有利」という想定の歪みが、勝率に出ているか確認したい）
3. その後 `play.py`（ホットシート）と簡単な tests、README を整える。

数値は全部 config で1行ずつ変えて再シミュレーションできる状態を最優先に。
